Retrieval Approach Comparison

Approach Strength Weakness Used Here
BM25 (Keyword) Exact term matching, fast, no model needed Misses synonyms and semantic meaning ✓
Vector Search (Semantic) Understands meaning, handles paraphrasing Can miss exact terms, embedding quality dependent ✓
Hybrid (RRF) Best of both — keyword precision + semantic recall Slightly higher latency from dual retrieval ✓
Cross-Encoder Reranking High-precision reordering of candidates Slower than bi-encoder, runs on top-N only ✓
Naive RAG (embed + retrieve) Simple to implement Lower recall, no precision refinement ✗
